# 🚨 紧急修复：云函数超时问题

## ❌ 错误信息

```
errCode: -504003 | errMsg: Invoking task timed out after 3 seconds
```

## 🔍 问题原因

**云函数执行超时**：AI 需要更多时间思考（特别是面对复杂问题时），但云函数的超时时间设置太短。

**当前设置**：3 秒（微信云开发默认值）  
**需要设置**：至少 30 秒，建议 60 秒

---

## ✅ 解决方案（3 种方法）

### 方法 1：通过微信开发者工具设置（推荐）⭐

#### 步骤：

1. **打开微信开发者工具**

2. **进入云开发控制台**
   - 点击顶部工具栏的 **"云开发"** 按钮

3. **找到云函数**
   - 点击左侧菜单的 **"云函数"**
   - 在列表中找到 `intentClassify` 函数

4. **修改超时时间**
   - 点击 `intentClassify` 进入详情页
   - 找到 **"高级设置"** 或 **"配置"** 标签页
   - 找到 **"超时时间"** 设置项
   - 将超时时间从 **3 秒** 改为 **60 秒**
   - 点击 **"保存"** 或 **"应用"**

5. **验证修改**
   - 返回云函数列表
   - 确认 `intentClassify` 的超时时间显示为 60 秒

---

### 方法 2：通过 config.json 设置

#### 步骤：

1. **打开文件**：`cloudfunctions/intentClassify/config.json`

2. **修改配置**：
```json
{
  "permissions": {
    "openapi": []
  },
  "timeout": 60,
  "memorySize": 256
}
```

3. **重新部署**：
   - 右键点击 `intentClassify` 文件夹
   - 选择 **"上传并部署：云端安装依赖"**

⚠️ **注意**：某些版本的微信云开发可能不读取 config.json，建议优先使用方法 1。

---

### 方法 3：通过云开发 CLI 设置

如果你安装了微信云开发 CLI 工具：

```bash
# 登录
wx-cloudbase login

# 更新云函数配置
wx-cloudbase function:update intentClassify --timeout 60
```

## 📊 超时时间建议

| 场景 | 建议超时 | 说明 |
|------|---------|------|
| **简单问答** | 10 秒 | 如"绿萝几天浇水" |
| **意图识别** | 30 秒 | 需要理解上下文 |
| **复杂问题** | 60 秒 | 如"列举所有植物" |
| **AI 生成内容** | 60 秒 | 如生成植物档案 |

**推荐设置**：60 秒（覆盖所有场景）

## 🔧 其他可能的问题

### 问题 1：修改后仍然超时

**可能原因**：
- 云函数没有重新部署
- 修改的是错误的云函数
- 微信服务器缓存未更新

**解决方案**：
1. 确认修改的是 `intentClassify` 函数
2. 重新部署云函数
3. 清除微信开发者工具缓存
4. 等待 5-10 分钟让服务器更新

### 问题 2：超时时间无法修改

**可能原因**：
- 云开发版本过低
- 权限不足
- 免费版限制

**解决方案**：
1. 升级微信开发者工具到最新版
2. 确认账号有云函数管理权限
3. 考虑升级到付费版（如果有版本限制）

### 问题 3：修改后 AI 响应仍然慢

**可能原因**：
- GLM-4-Flash API 响应慢
- 网络延迟
- Prompt 过长

**解决方案**：
1. 检查云函数日志，确认 AI 响应时间
2. 优化 SYSTEM_PROMPT，减少冗余内容
3. 添加响应时间监控

## 📝 云函数日志查看方法

### 查看步骤：

1. **微信开发者工具** → **云开发控制台**

2. **云函数** → **intentClassify** → **日志**

3. **筛选日志**：
   - 时间范围：最近 1 小时
   - 错误级别：Error / Warning

4. **关键日志**：
```
[CloudAPI] Calling intentClassify...
[CloudAPI] intentClassify completed in XXXXms
```

如果看到 `completed in 3000+ms` 然后报错，说明超时时间设置成功，但 AI 思考时间确实很长。
