## 🌿 光照标签优化

### 问题
女王大人指出：花园列表页每个植物的光照标签太笼统，只显示"需阳光"，没有区分：
1. **光照类型**：直射光/散射光/耐阴/半阴
2. **温度要求**：具体温度范围（如 15-25℃）

### 解决方案

#### 1. 云函数 API 增强
**文件：** `cloudfunctions/getCareGuide/index.js`

- 在 prompt 中新增 `lightType` 字段（必填，枚举值：'直射光' | '散射光' | '耐阴' | '半阴'）
- 在 prompt 中新增 `tempRange` 字段（必填，对象格式：`{ min: 15, max: 25, unit: '℃' }`）
- 更新 `getDefaultPlant()` 默认值，包含这两个字段

#### 2. 服务层数据处理
**文件：** `services/garden.js`

- 在 `processPlantData()` 中提取 `lightType` 和 `tempRange`
- 添加 `getLightTypeFromText()` 辅助函数，从光照描述文本中推断光照类型（降级方案）
- 优先级：`identifyResult.careGuide.lightType` > `careGuide.lightType` > 从文本推断

#### 3. 列表页显示优化
**文件：** `pages/my-plants/my-plants.wxml`

显示格式：`[图标] 光照类型 · 温度范围`
- 直射光 → ☀️
- 散射光 → 💡
- 耐阴 → 🌑
- 半阴 → 🌤️

示例：`☀️ 直射光 · 15-25℃`

### 修改清单

| 文件 | 修改内容 |
|------|----------|
| `cloudfunctions/getCareGuide/index.js` | 新增 `lightType` 和 `tempRange` 字段 |
| `services/garden.js` | 新增数据处理和降级推断逻辑 |
| `pages/my-plants/my-plants.wxml` | 显示光照类型图标 + 温度范围 |
| `pages/my-plants/my-plants.wxss` | 优化状态行样式 |

### 注意事项

1. **云函数需要重新部署**：修改后需在微信开发者工具中上传并部署 `getCareGuide` 云函数
2. **已有植物数据**：旧植物的 `identifyResult.careGuide` 没有这两个字段，会走降级推断逻辑
3. **新识别植物**：会直接从 AI 返回结构化数据

### 后续优化

- 可在植物详情页也显示结构化的光照和温度信息
- 可添加光照类型的筛选功能（如：只显示耐阴植物）

## 🖼️ 搜索结果页图片源优化

### 问题
女王大人指出：搜索结果页没有头图，需要**免费且有数量保证**的图片源

### ⚠️ 重要教训

**教训 1：** PlantNet 是国外服务器，不稳定！（之前 GBIF 已吃过亏）

**教训 2：** 腾讯云 COS 不合理：
- ❌ 需要自己下载上传所有图片，工作量巨大
- ❌ 免费期只有 180 天，之后收费
- ❌ 本末倒置，为了免费图片花大量时间维护

### ✅ 最终方案：简化的多源降级策略

**优先级：** GBIF（云函数端）> 维基百科 > 默认图保底

#### 1. GBIF（首选）⭐⭐⭐⭐
- **API：** 物种搜索 + 图片获取（两步）
- **优点：** 云函数服务器访问、图片数量极多、完全免费
- **关键：** 云函数在海外服务器，访问 GBIF 不受国内网络影响

#### 2. 维基百科中文版（备选）⭐⭐⭐
- **API：** `https://zh.wikipedia.org/api/rest_v1/page/summary/{plantName}`
- **优点：** 常见植物都有、一步获取
- **缺点：** 罕见植物可能没有

#### 3. 默认图片保底（保证不空白）⭐⭐
- **图片：** Unsplash 植物图库（3 张随机）
- **作用：** 所有源都失败时保底

### 修改清单

| 文件 | 修改内容 |
|------|----------|
| `cloudfunctions/getCareGuide/index.js` | 简化 `getPlantImage()`，移除 COS |
| `docs/ai-plant-butler/植物图片源解决方案.md` | 重写方案（只用免费 API） |

### 核心原则

1. **完全免费** - 无需 API Key，无需维护
2. **无需准备** - 不调用即取即用
3. **云函数端访问** - GBIF 在海外服务器访问快
4. **有保底** - 最差情况也有默认图
