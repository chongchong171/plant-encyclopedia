# 植物图片源解决方案

**问题：** 搜索结果页没有头图，需要免费且有数量保证的图片源

**原则：** 只用免费 API，不维护图片库，不依赖不稳定源

---

## ⚠️ 排除的方案

### ❌ 腾讯云 COS（不推荐）
- **原因：** 需要自己下载上传所有图片，工作量巨大
- **免费期：** 只有 180 天，之后收费
- **结论：** 本末倒置，为了免费图片花大量时间维护，不合理

### ❌ PlantNet（不推荐）
- **原因：** 法国服务器，国内访问不稳定
- **教训：** 之前 GBIF 已经吃过亏了

---

## ✅ 推荐方案：多源降级策略

**优先级：** GBIF（云函数端）> 维基百科 > 默认图保底

```
1. GBIF（云函数服务器访问）
   ↓ 失败
2. 维基百科中文版
   ↓ 失败
3. 默认植物图片（Unsplash，保证不空白）
```

### 核心优势

1. **完全免费** - 无需 API Key，无需维护
2. **无需准备** - 不调用即取即用
3. **云函数端访问** - GBIF 在海外服务器访问快
4. **有保底** - 最差情况也有默认图

---

## 📊 各图片源详情

### 1. GBIF（首选）⭐⭐⭐⭐

**API：**
- 物种搜索：`https://api.gbif.org/v1/species/match?name={plantName}`
- 图片获取：`https://api.gbif.org/v1/occurrence/search?taxonKey={speciesKey}&mediaType=StillImage`

**优点：**
- 全球最大生物多样性数据库
- 图片数量极多（数亿张）
- **云函数服务器访问，不受国内网络影响**
- 完全免费，无需 API Key

**缺点：**
- 需要两步查询（先物种后图片）
- 国内直接访问慢（但云函数端没问题）

**适用：** 绝大多数植物查询

---

### 2. 维基百科中文版（备选）⭐⭐⭐

**API：** `https://zh.wikipedia.org/api/rest_v1/page/summary/{plantName}`

**优点：**
- 常见植物都有词条和图片
- API 简单，一步获取
- 中文维基百科，相对稳定

**缺点：**
- 只有常见植物有图片
- 罕见植物可能没有词条

**适用：** GBIF 失败时的降级方案

---

### 3. 默认图片（保底）⭐⭐

**图片：** Unsplash 植物图库（3 张随机）

**优点：**
- 保证页面不空白
- 图片质量高
- 无需维护

**缺点：**
- 不是对应植物的真实图片

**适用：** 所有源都失败时的保底
