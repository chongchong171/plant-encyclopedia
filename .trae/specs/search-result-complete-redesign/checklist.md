# Checklist

## 结构重构验证
- [ ] 植物头图模块正确显示（图片、收藏按钮、提示文字）
- [ ] 基本信息模块包含：中文名、拉丁学名、别名、科属、养护难度
- [ ] 植物档案模块包含：形态特征、观赏价值、原产地
- [ ] 生长习性模块包含：生长环境描述
- [ ] 主要价值模块包含：观赏价值、净化能力、药用食用、寓意
- [ ] 养护指南模块包含 8 项：光照、浇水、温度、湿度、施肥、土壤、修剪、繁殖
- [ ] 养护要点模块显示 4 条核心要点
- [ ] 季节性养护模块可折叠（如有数据）
- [ ] 植物趣闻模块可折叠（如有数据）
- [ ] 已删除旧的不必要的卡片（植物价值、栽培建议等）

## 视觉设计验证
- [ ] 植物头图样式美观（图片圆角、阴影、收藏按钮位置）
- [ ] 基本信息卡片设计清晰（名称突出、难度标签醒目）
- [ ] 所有卡片风格统一（圆角、阴影、间距）
- [ ] 养护指南网格布局合理（自适应、不拥挤）
- [ ] 8 项养护内容有独特的颜色标识（光照黄色、浇水蓝色等）
- [ ] 文字排版清晰（字号适中、行距合理、间距舒适）
- [ ] 响应式设计生效（小屏幕、大屏幕、横屏模式）
- [ ] 动画效果流畅（卡片悬停、折叠展开）

## 功能验证
- [ ] 页面正常加载，无错误
- [ ] 条件显示的内容正确（有数据才显示对应模块）
- [ ] 折叠/展开功能正常（季节性养护、植物趣闻）
- [ ] 收藏功能正常（点击收藏、取消收藏）
- [ ] 图片预览功能正常（点击图片放大）
- [ ] 难度星级正确显示（根据难度等级生成）
- [ ] 页面性能良好（加载速度快、渲染流畅）

## 内容验证
- [ ] 植物档案内容 80-120 字左右
- [ ] 生长习性内容 60-80 字左右
- [ ] 主要价值内容 50-70 字左右
- [ ] 养护指南每项 25-35 字左右
- [ ] 养护要点每条 20 字以内，最多 4 条
- [ ] 养护指南内容可操作（有具体数字和方法）
- [ ] 养护指南内容有区分度（不同植物养护方法有差异）
- [ ] 使用通俗语言，少用专业术语

## 代码质量验证
- [ ] WXML 结构清晰，标签语义化
- [ ] WXSS 样式规范，无重复代码
- [ ] JavaScript 逻辑清晰，无冗余代码
- [ ] 无控制台错误或警告
- [ ] 代码符合微信小程序规范
