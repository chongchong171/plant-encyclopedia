# 🚀 云函数部署指南

## ⚠️ 重要提示

**修改云函数代码后，必须在微信开发者工具中重新部署才能生效！**

---

## 📋 部署步骤

### 方法 1：右键部署（推荐）

1. 打开**微信开发者工具**
2. 在左侧文件树中找到 `cloudfunctions/intentClassify` 文件夹
3. **右键点击** `intentClassify` 文件夹
4. 选择 **"上传并部署：云端安装依赖"**
5. 等待部署完成（状态栏显示"上传成功"）

### 方法 2：工具栏部署

1. 打开**微信开发者工具**
2. 点击顶部工具栏的 **"云开发"** 按钮
3. 进入云开发控制台
4. 选择 **"云函数"** 标签页
5. 找到 `intentClassify` 函数
6. 点击 **"上传部署"**

## 🔧 如果部署后仍然无效

### 可能原因 1：缓存问题

**解决方案**：
1. 在微信开发者工具中，点击 **"清缓存"**
2. 勾选 **"清除全部缓存"**
3. 重新编译项目
4. 重新部署云函数

### 可能原因 2：云函数环境不一致

**解决方案**：
1. 确认云函数部署的环境（开发版/体验版/正式版）与小程序一致
2. 在 `project.config.json` 中检查 `cloudfunctionRoot` 配置
3. 在云函数代码中确认 `cloud.init` 使用的环境

### 可能原因 3：Prompt 过长

如果 Prompt 超过模型限制，可能导致部署失败或被截断。

**检查方法**：
```bash
# 在云函数目录执行
cd cloudfunctions/intentClassify
node -e "console.log(require('fs').readFileSync('index.js', 'utf8').length)"
```

**解决方案**：
- 如果文件超过 5000 行，需要精简 Prompt
- 删除冗余的示例，保留核心策略

## 🆘 常见问题

### Q: 部署后还是回复"AI 想太久了"怎么办？

A: 
1. 确认云函数已成功部署（查看部署日志）
2. 检查云函数超时时间设置（建议 60 秒）
3. 查看云函数运行日志，确认是否报错
4. 尝试简化 Prompt，减少示例数量

### Q: 如何查看云函数日志？

A:
1. 微信开发者工具 → 云开发控制台
2. 云函数 → intentClassify → 日志
3. 查看最近的运行日志，确认是否有错误

### Q: 真机调试和开发工具中表现不一致？

A:
1. 确认云函数部署的环境与小程序一致
2. 清除小程序缓存
3. 重新编译并上传代码
4. 在体验版/开发版中测试
