# 优化养护指南排版 - 部署指南

## 问题描述

养护指南的"环境要求"部分总是出现单字、少字换行的情况：

```
❌ 问题示例：
🌡️ 温度：适宜温度 15-28℃，冬季注意保
   暖  ← 2 个字

💦 湿度：喜欢湿润环境，可定期向叶片喷
   水  ← 1 个字

☀️ 光照：喜充足光照，避免强烈直射阳
   光  ← 1 个字
```

**影响**：排版不美观，阅读体验差。

## 问题原因

AI 在生成养护指南时，没有考虑移动端显示的换行问题：
- 按照正常语义生成内容
- 没有控制每行字数
- 导致文字到边界时强制换行，出现单字换行

## 修复内容

### 更新 `intentClassify` 云函数

**文件**: `cloudfunctions/intentClassify/index.js`

**修改**: 在 SYSTEM_PROMPT 中添加【养护指南格式要求】

**核心规则**:
1. **每行字数控制**: 每条养护说明控制在 14-18 个字以内
2. **避免单字换行**: 调整措辞，避免在句子最后一个字换行
3. **换行优化**: 如果必须换行，在第一句话中间换行，确保换行后至少 2-3 个字

**示例对比**:

❌ **错误示范**（避免单字/少字换行）：
```
"冬季注意保\n暖"（2 字换行）
"可定期向叶片喷\n水"（1 字换行）
"避免强烈直射阳\n光"（1 字换行）
```

✅ **正确示范**（调整措辞，避免换行）：
```
"冬季需注意保暖"（不换行）
"可定期向叶片喷水"（不换行）
"避免强烈阳光直射"（不换行）
```

**如果内容较长，必须换行时的示例**:

✅ **正确**:
```
"保持土壤湿润，见\n干见湿，避免积水"
```

❌ **错误**:
```
"保持土壤湿润，见干见湿，避免\n积水"
```

## 部署步骤

### 步骤：上传并部署 `intentClassify` 云函数

1. 打开微信开发者工具
2. 右键点击 `cloudfunctions/intentClassify` 目录
3. 选择 **"上传并部署：云端安装依赖"**
4. 等待部署完成（状态栏显示"上传成功"）
5. 版本号：`v5.5-optimize-care-guide-format-2026-04-17`

1. 打开小程序，进入首页
2. 展开 AI 对话框

## 回滚方案

如果部署后出现问题：

1. 右键点击 `cloudfunctions/intentClassify`
2. 选择 **"回滚版本"**
3. 选择之前的版本（v5.4-use-deletePlant-tool-2026-04-17）进行回滚

## 注意事项

1. **依赖安装**：确保选择"云端安装依赖"选项
2. **测试环境**：建议先在测试环境验证，再发布到正式环境
3. **AI 响应时间**：由于增加了格式要求，AI 响应时间可能略微增加（约 1-2 秒）
4. **格式一致性**：AI 会尽量遵守格式要求，但可能偶尔仍有不完美的情况，这是正常的
