## 一、当前数据基础

`analytics_track` 云函数已在运行，自动收集三类数据：

| 集合 | 用途 | 已有字段 |
|------|------|---------|
| `analytics_users` | 用户档案 | registerDate, lastVisit, totalVisits, visitDates, actions(识别/诊断/养护/添加/收藏), totalSessions, avgSession |
| `analytics_daily` | 每日汇总 | date, newUsers, activeUsers, returningUsers, apiCalls, funnel(visit→identify→addPlant→favorite), totalSessions |
| `analytics_events` | 原始事件流 | openId, event, timestamp, duration, success, extra |

前端埋点入口（`app.js`）：
- `user_visit` — 用户访问
- `session_end` — 会话结束（含 duration）
- `identify_plant` / `get_care_guide` / `diagnose_plant` / `add_plant` / `favorite_plant`

## 三、技术实现方案

### 3.1 新建云函数

**`cloudfunctions/getAnalytics/index.js`**

职责：
1. 校验管理员身份（复用 `updateAppConfig` 的 admin 校验逻辑）
2. 读取 `analytics_daily` 最近 N 天数据
3. 读取 `analytics_users` 聚合统计（总用户数、今日新客等）
4. 读取 `analytics_events` 最近 20 条
5. 返回结构化数据给前端

入参：
```js
{
  type: 'overview' | 'trend' | 'funnel' | 'events',
  days: 7  // 趋势图天数，默认 7
}
```

出参：
```js
{
  success: true,
  overview: { todayVisits, todayNewUsers, todayIdentify, avgSession },
  trend: [
    { date: '2026-05-03', visits: 120, newUsers: 10, returningUsers: 30, identify: 80, diagnose: 20, careGuide: 40 },
    // ...
  ],
  funnel: { visit: 1000, identify: 650, addPlant: 280, favorite: 180 },
  topFeatures: [
    { name: '拍照识别', count: 1234 },
    { name: '植物诊断', count: 567 },
    // ...
  ],
  recentEvents: [
    { time: '14:32', openId: 'oP55****', event: 'identify_plant', extra: {} },
    // ...
  ]
}
```

### 3.2 前端页面

**`pages/admin/dashboard.wxml` + `dashboard.js` + `dashboard.wxss`**

作为管理控制台的一个子页面或 Tab：

- 在 `admin.wxml` 顶部增加 Tab 切换：「配置管理」/「数据看板」
- 数据看板页面内部分区块展示上述 5 个模块
- 使用微信小程序原生图表库（如 `ec-canvas` ECharts）或纯 CSS 实现简易图表

**建议图表方案**：
- 趋势图：引入 `echarts-for-weixin`（`ec-canvas` 组件）
- 漏斗图：CSS 梯形或简单条形图
- 卡片/排行/事件流：纯 WXML + WXSS

### 3.3 需要修改的文件

| 文件 | 修改内容 |
|------|---------|
| `pages/admin/admin.wxml` | 顶部增加 Tab 切换栏（配置 / 数据） |
| `pages/admin/admin.js` | 增加 `currentTab` 状态，控制显示配置页面或数据看板 |

| `cloudfunctions/getAnalytics/package.json` | 新建，依赖 `wx-server-sdk` |

### 3.4 引入 ECharts（可选）

如果选择用 ECharts 画趋势图：

1. 下载 `echarts-for-weixin` 到 `components/ec-canvas/`
2. 在 `pages/admin/dashboard.json` 中声明组件
3. 在 `dashboard.wxml` 中使用 `<ec-canvas>`

如果希望轻量，也可以用纯 CSS 画柱状图/折线图，避免引入外部库。

## 五、注意事项

- `analytics_events` 数据量会很大，查询时务必 `limit(20)` 并建立索引
- `analytics_daily` 按日期聚合，查询最近 7 天即可，无需全量读取
- 云函数返回数据建议做 openId 脱敏（如 `oP55****`），避免在管理界面泄露用户隐私
- 如果用户量增大，`getAnalytics` 可能需要分页或增加缓存
