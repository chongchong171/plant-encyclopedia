## ⚠️ 风险评估

### 🔴 高风险问题

#### 1. **维基百科降级缺失**
- **问题：** 常见植物（芦荟、绿萝、吊兰等）维基百科都有高质量图片
- **影响：** GBIF 和 Pexels 都失败时，维基百科可以成功，但代码无法获取
- **历史教训：** 2026-04-12 文档规划了 GBIF → 维基百科 → 默认图三级降级，但实际只实现了 GBIF → Pexels
- **结果：** 用户体验损失（常见植物反而可能没有图片）

#### 2. **缺少保底机制**
- **现状：** 所有源都失败时返回 `null`
- **影响：** 页面显示空白或占位符
- **历史教训：** 2026-04-11 正是因为没有降级方案导致头图缺失

#### 3. **维基百科降级方案缺失**
- **文档规划：** GBIF → 维基百科 → 默认图
- **实际实现：** GBIF → Pexels（未启用）
- **风险：** 常见植物（如芦荟、绿萝）维基百科都有图片，但代码没有利用

### 🟡 中风险问题

#### 3. **GBIF 成功率依赖拉丁学名**
- **问题：** AI 可能返回错误的拉丁学名或不返回
- **历史教训：** 2026-04-11 出现过 `scientificNameLatin` 为空的情况
- **当前措施：** Prompt 中强调必填，但无法 100% 保证

#### 4. **Pexels 中文名搜索效果**
- **问题：** Pexels 是英文图片库，用中文名搜索可能不准确
- **示例：** 搜索"芦荟" vs "Aloe vera"
- **影响：** 即使配置了 API Key，效果可能也不理想

#### 5. **环境变量配置确认**
- **问题：** Pexels API Key 已配置，但需要确认云函数环境变量是否生效
- **影响：** 如果环境变量未设置，代码中的 `process.env.PEXELS_API_KEY` 会返回空字符串
- **建议：** 在云开发控制台确认
