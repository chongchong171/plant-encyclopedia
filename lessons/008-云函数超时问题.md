# 教训 008 - 云函数超时问题

**日期：** 2026-04-18  
**级别：** ⭐⭐

## 问题描述
云函数响应时间过长（25 秒），导致前端超时

## 原因分析
- 串行调用多个外部 API（PlantNet + 百度 AI + GLM）
- 单个 API 失败会阻塞整体流程
- 没有设置合理的超时时间

## 解决方案
- 并行调用多个 API（Promise.all）
- 使用 Promise.race 设置超时
- 前端超时 22 秒，云函数超时 25 秒

## 预防措施
- 外部 API 调用必须设置超时
- 使用 Promise.race 防止单个 API 卡住
- 准备降级方案（API 失败时使用预设数据）
